Where should uploaded images and data be stored?

This decides what free accounts you'll need to create

A

Supabase — Free Recommended

One free account at supabase.com handles everything — database (team, services, blog, clients, contact) and image file storage. Industry standard, very reliable, free up to 500MB storage and 50,000 rows.

✓ One account for everything ✓ Images + database together ✓ Free forever for small sites
B

Cloudinary (images) + JSON files (data)

Images uploaded to Cloudinary (free image CDN with auto-resize). All other data (team, blog, services) saved as JSON files in the project. Simple but data is stored in code — needs a redeploy to persist across server restarts.

⚠ Data lost on redeploy ✓ Great image optimization
C

Local only (no external accounts)

Everything saved directly on the server — images to public/uploads/, data to JSON files. Works with no accounts. Fine if the website runs on a dedicated server, but changes are lost if the hosting platform resets the file system (common on Vercel/Netlify).

✗ Lost on Vercel/Netlify deploy ✓ No accounts needed
