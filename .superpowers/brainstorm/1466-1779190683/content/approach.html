How should the admin dashboard work?

Three approaches to content management — pick the one that fits best

A

Built-in Admin Panel Recommended

A password-protected /admin page built directly into your website. You log in, edit team members, images, services and blogs through a clean form UI. Data stored in a free Supabase database. No external tools needed — everything lives in your site.

Pros

  • No external accounts or tools
  • 100% custom to DOK's content
  • Works offline for preview
  • Free to run

Cons

  • Takes 2–3 days to build
  • We build the UI from scratch
B

Sanity CMS Studio

Connect a professional headless CMS called Sanity. You manage all content in Sanity's own beautiful dashboard at sanity.io/manage — images, team, services, blogs. Changes publish to the website instantly without redeploying.

Pros

  • Professional CMS built for this
  • Real-time publish, no deploy
  • Image optimization built in
  • Free tier available

Cons

  • External tool to learn
  • Requires significant refactoring
  • Paid plan if traffic grows
C

Notion as CMS

Use Notion databases as your content source. Add a team member row in Notion → it appears on the website. Add a blog post page in Notion → published automatically. Uses the free Notion API.

Pros

  • Familiar Notion interface
  • Free to use
  • Quick to set up

Cons

  • Notion API can be slow
  • Limited image handling
  • Content tied to Notion account
